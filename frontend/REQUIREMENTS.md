# 打了吗 - 前端需求文档

## 1. 项目概述

微信小程序**"打了吗"** —— 一款面向好兄弟的"健康管理"工具。

> 今天，你冲了吗？

群友每次"手冲"后点一下打卡，系统自动统计月度数据，月底评选**飞机王**👑。项目本质是群聊里的整活玩具，画风诙谐、文案骚气、但数据统计该认真一点不含糊。

**核心原则**：
- 打开即冲，冲完即走，操作成本为 0
- 卷王和摸鱼怪，数据面前无所遁形
- 月底飞机王，必须 @ 出来公开处刑

## 2. 技术选型

**选用 uni-app（Vue3）**。

理由：

| 考虑点 | 说明 |
|--------|------|
| **留后路** | 内容尺度原因，微信提审可能不过。uni-app 同一套代码可编译为 H5，直接丢个 URL 到群里就能用，不依赖任何平台审核 |
| **渐进策略** | 先用微信体验版跑，被封/想扩大 → 切 H5 零成本迁移 |
| **开发效率** | Vue3 语法上手快，uView UI 组件丰富 |
| **性能足够** | 这种打卡工具没有复杂渲染，uni-app 性能完全够 |

**编译目标优先级**：
1. 微信小程序（体验版，免审核，群友扫码用）
2. H5（备选后路，部署到服务器，分享链接即用）
3. App（远期可选，打包 iOS/Android）

**配套依赖**：
- uView UI 3.0（Vue3 组件库）
- dayjs（日期处理）
- pinia（状态管理，管理当前用户、当前战队等全局状态）

## 3. 核心设计：群组

> **群组是整个产品的核心容器**。打卡、排行、飞机王都按群独立计算。一个用户可以加入多个群，切换群不影响各群的打卡数据。

### 3.1 用户旅程

```
新兵入伍：
  起花名 → 建战队 / 扫码入队 → 冲！

老兵归队：
  自动识别身份 → 切到上次的战队 → 继续冲

转战：
  切战队 → 各群数据独立，A群卷王不妨碍B群摸鱼
```

### 3.2 用户身份方案

```
首次打开：
  1. 本地生成 UUID
  2. 填昵称 → POST /api/user { user_id, nickname }
  3. 创建或加入群 → POST /api/group 或 /api/group/join
  4. user_id + active_group_id 存入本地 storage

再次打开：
  1. 从本地 storage 读取 user_id
  2. GET /api/user/:id/groups → 获取群列表
  3. 用户不存在 → 重新走首次流程
  4. 群列表为空 → 引导创建/加入群
```

## 6. 数据流

### 6.1 新兵入伍

```
[小程序]                        [后端]
  │                               │
  │── 生成 UUID                   │
  │── 本地存 user_id              │
  │── 起花名                      │
  │── POST /api/user ────────────→│── 查无此人，创建新兵档案
  │   { user_id, nickname }       │
  │←── { code:0, user } ─────────│
  │── 建队/入队 → 开冲！          │
```

### 6.2 老兵归队

```
[小程序]                        [后端]
  │                               │
  │── 读本地 user_id              │
  │── GET /api/user/:id/groups ──→│── 查出所有已加入的战队
  │←── 返回战队列表               │
  │── 选上次的战队 → 进入战场      │
```

### 6.3 冲刺

```
[首页]                        [后端]
  │                               │
  │── 点"冲"按钮                 │
  │── 防抖 1s（防止连点）         │
  │── POST /api/checkin ─────────→│── 写入战斗记录
  │   { user_id, group_id }      │
  │←── { code:0, count: 5 } ─────│
  │── 🛩️ 冲刺动画               │
  │── wx.vibrateShort() 震动     │
  │── Toast 随机骚话             │
  │── 刷新战况                    │
```

## 9. 后路策略

当微信小程序因内容审核无法上架时，切换流程：

```
1. 修改 manifest.json → 编译目标改为 H5
2. uni-app build → 生成 H5 静态文件
3. 部署到服务器，绑定域名
4. 群里发链接 → 添加到手机桌面 → 体验接近原生 App
```

H5 模式下功能无损：
- `uni.setStorageSync` → 浏览器 localStorage
- `uni.request` → 浏览器 fetch
- 震动 API 不可用 → 降级为视觉动画
- 微信分享不可用 → 降级为复制链接
